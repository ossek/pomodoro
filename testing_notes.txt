jasmine wiki:

beforeEach -- like setup from xUnit

module -- simply loads the module

addMatchers -- see http://docs.angularjs.org/tutorial/step_11
  https://github.com/pivotal/jasmine/wiki/Matchers
  - the first link describes why it is necessary to add a 'toEqualData' matcher ( namely the $resource service augments response object with some additional stuff

describe -- setup a scope for tests

inject -- for mocking
 http://docs.angularjs.org/api/angular.mock.inject
 specifically, tells the injector to inject the functions you pass inject

underscore wrapping is important to avoid variable hiding:
  http://docs.angularjs.org/api/angular.mock.inject

module function (often used in beforeEach)
  http://docs.angularjs.org/api/angular.mock.module
  registers module configuration info to be used when injector is
  created by inject

$compile -- http://docs.angularjs.org/api/ng.$compile
  compiles HTML string or DOM into a template and produces template function.
  template function used to link scope and template together. i.e., 
  the function takes a scope as a parameter like:
  
  var element = $compile('<span countdown=0></span>')($rootScope); 

for a directive that manipulates the DOM, it seems that, for the scope supplied to
$compile, we must call $digest.  e.g.
https://github.com/vojtajina/ng-directive-testing/blob/start/test/tabsSpec.js
or
$rootScope.$digest();
